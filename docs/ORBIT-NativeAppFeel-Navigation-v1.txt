================================================================================
  PART 2 — THE ROOT CAUSE
  (Why ORBIT feels like a website, not an app)
================================================================================

  Yeh sirf design ka sawaal nahi hai. Yeh FEELING ka sawaal hai.

  ╔════════════════════════════════════════════════════════════════════╗
  ║  WEBSITE FEEL =                                                    ║
  ║  Too many navigation options + Section-header-heavy layout +       ║
  ║  Padded cards that breathe like web articles +                     ║
  ║  Too much color = information overload +                           ║
  ║  No gestures, no swipe, no physics =                               ║
  ║  Brain registers: "I'm browsing a website"                         ║
  ╠════════════════════════════════════════════════════════════════════╣
  ║  APP FEEL =                                                        ║
  ║  4 clear destinations + Tight list density +                       ║
  ║  Surfaces feel physical (cards you can swipe) +                    ║
  ║  One accent, used sparingly =                                      ║
  ║  Brain registers: "I'm using an app"                               ║
  ╚════════════════════════════════════════════════════════════════════╝

  THE 5 SPECIFIC REASONS ORBIT FEELS LIKE A WEBSITE:

  REASON 1: 6 TABS (Websites have navbars with 5-7 items. Apps have 4.)
  REASON 2: SECTION HEADERS EVERYWHERE ("GROUP ROOMS", "MOOD ROOMS",
            "WEEKLY CHALLENGES", "SPOTLIGHT FEED" — all caps labels =
            webpage section dividers, not app UI)
  REASON 3: CARDS WITH TOO MUCH PADDING (Website article cards breathe.
            App list items are dense and scannable.)
  REASON 4: EVERY SECTION HAS A DIFFERENT COLOR LOGIC (Neon purple cards,
            yellow badges, blue pills, orange LEGEND — visual chaos =
            designer trying too hard = website aesthetic)
  REASON 5: NO NATIVE GESTURES VISIBLE IN DESIGN (Where is swipe-to-reply?
            Where is swipe-to-dismiss? Apps have physics. Websites don't.)



================================================================================
  PART 3 — THE 4-TAB NAVIGATION SYSTEM
================================================================================

  DECISION: ORBIT will have EXACTLY 4 tabs. No more. No less.

  Telegram ka bottom nav: Chats | Contacts | Settings | Profile (4 tabs)
  ORBIT ka bottom nav:    Rooms | Discover  | Ranks    | You     (4 tabs)

  ── 3.1 · WHAT HAPPENED TO BAZAAR? ──────────────────────────────────────────

  Bazaar (skill marketplace) does NOT deserve a permanent bottom tab.
  Reason: Bazaar is a UTILITY, not a DESTINATION. Users go there
  occasionally to post/find gigs — not as a daily habit.

  Solution: Bazaar moves to Discover tab as a section.
    Discover tab → top pill filters: [Hot 🔥] [Near You] [Challenges] [Bazaar]
    Or: Bazaar accessible from Rooms screen via a top-right icon.
    Either way: NOT a permanent bottom tab.

  ── 3.2 · THE 4-TAB STRUCTURE ────────────────────────────────────────────────

  TAB 1 — ROOMS (chat bubble icon)
    What's here:
      • All your rooms (conversations you're in)
      • Spotlight Live pinned at top (inline, not a separate banner)
      • DMs section below rooms
    Icon: MessageSquare (Lucide)
    Badge: Unread count (only if >0, max shows "99+")

  TAB 2 — DISCOVER (compass/explore icon)
    What's here:
      • Mood Rooms (horizontal scroll)
      • Hot Rooms feed (vertical list)
      • Weekly Challenges
      • Bazaar (accessible via top pill filter)
      • "For You" personalized feed (from ZeroFriction prompt)
    Icon: Compass (Lucide)
    Badge: None (or dot if new challenge started)

  TAB 3 — RANKS (trophy/medal icon)
    What's here:
      • Global leaderboard
      • Weekly leaderboard
      • Challenge leaderboard
      • Your position (always visible at bottom of list, sticky)
    Icon: Trophy (Lucide)
    Badge: None

  TAB 4 — YOU (person/circle icon)
    What's here:
      • Your profile + ORBIT Card
      • Trust Score
      • Stats (Karma, Posts, Watches, Credits)
      • Settings access
      • Cashout access
    Icon: CircleUser (Lucide)
    Badge: None

  ── 3.3 · BOTTOM NAV COMPONENT SPEC ─────────────────────────────────────────

  Height: 64px (not including safe area / home indicator)
  Background: --bg-base with 1px --border-subtle top border
  NO backdrop blur on bottom nav (adds lag on low-end Android)

  Each tab item:
    Width: 25% of screen
    Touch target: 44×44px minimum
    Layout: icon (22px) on top, label (11px, font-weight 500) below
    Gap between icon and label: 4px

  INACTIVE state:
    Icon: --text-tertiary (#6B6B73)
    Label: --text-tertiary
    Background: transparent

  ACTIVE state:
    Icon: --accent (#5B7FFF) — filled variant if available, else stroke
    Label: --accent, font-weight 600
    Background: transparent (NO pill, NO box, NO underline)
    Top indicator: 2px line at very top of tab area, --accent color,
                   centered, 24px wide. This is the ONLY active indicator.

  Unread badge (Rooms tab only):
    8px × 8px red dot (--danger) when unread exists
    OR "12" inside 18px circle when count > 1
    Positioned: top-right of icon, -4px offset
    NEVER show badge on Discover, Ranks, or You tabs.

  ── 3.4 · CREDIT DISPLAY (Permanent Location) ───────────────────────────────

  v2 prompt mein credits ka location inconsistent tha (only on Discover).
  Fix: Credits are always visible in a SINGLE consistent place.

  Location: App-level persistent top pill, visible on ALL screens.
  BUT — not in the app bar (too crowded). Instead:
    On Rooms/Discover/Ranks: credits pill appears subtly under app bar title.
    On You (Profile): credits are in the stats row.

  OR — simplest solution (recommended):
    Credits pill moves to the "You" tab only.
    On other screens: NOT visible. (Clean is better than consistent-but-noisy.)
    User who wants to check credits → taps "You" tab.
    This reduces visual noise on Rooms/Discover/Ranks screens.

  ── 3.5 · NAVIGATION BEHAVIOR ────────────────────────────────────────────────

  Tab switch animation: content fades (120ms out, 180ms in). No slide.
  Back navigation: System back button / swipe-from-left.
  Double-tap tab: Scroll to top of that screen (like Instagram/Telegram).
  Long-press tab: No action (don't add complexity).



================================================================================
  PART 4 — APP-FEEL RULES
  (The 12 things that make ORBIT feel native, not web)
================================================================================

  RULE AF-01 · LIST DENSITY IS SACRED
    App lists are DENSE. Standard room list item: 68px tall.
    (Telegram's chat items are ~72px. Keep it close.)
    Website card: 120px+ with lots of padding. Kill that.
    Every list item: 14px horizontal padding on content, 14px vertical.

  RULE AF-02 · SECTION HEADERS ARE MINIMAL
    Replace all-caps section headers with:
      → Nothing (let context speak), OR
      → A small 11px overline label, --text-tertiary, no bold, no tracking
    "GROUP ROOMS" → delete it
    "MOOD ROOMS" → tiny overline, or remove in favor of visual context
    "WEEKLY CHALLENGES" → overline "This Week" max

  RULE AF-03 · SWIPE GESTURES EVERYWHERE
    Room list item: swipe LEFT → mute/leave options
    Room list item: swipe RIGHT → quick-reply
    Message in chat: swipe RIGHT → reply (like Telegram/WhatsApp)
    These don't need to be built tomorrow — but the DESIGN must show
    the intention (visual hint: subtle edge gradient when dragging)

  RULE AF-04 · BOTTOM SHEET OVER MODAL
    Any action that requires a decision: use a bottom sheet (slides up).
    NOT a centered modal.
    NOT a full-screen push.
    Bottom sheets feel native. Centered modals feel web.
    Bottom sheet: 24px top radius, drag handle, backdrop blur 8px.

  RULE AF-05 · HAPTIC FEEDBACK INTENT
    Design should note where haptic feedback fires:
      → Button press: light impact
      → Message send: light tap
      → Credits received: medium impact
      → Badge earned: heavy impact
    Haptics = physical = native app feeling.

  RULE AF-06 · SKELETON LOADING, NOT SPINNERS
    Every list that loads shows skeleton cells (gray shimmer rectangles
    where content will appear). NEVER a centered spinner on a list.
    Spinner = web. Skeleton = app.

  RULE AF-07 · STICKY HEADERS, NOT FLOATING ONES
    App-bar sticks to top as user scrolls. Content scrolls under it.
    The app-bar has a 1px --border-subtle bottom that appears when
    content is under it (scroll-activated border — like iOS/Apple).

  RULE AF-08 · PULL-TO-REFRESH IS STANDARD
    Every list screen has pull-to-refresh.
    Custom indicator: 20px Lucide RefreshCw icon, rotates with pull,
    subtle spring when released. NOT a loading bar at top.

  RULE AF-09 · EMPTY STATES ARE WARM, NOT BLANK
    Empty room list: icon + "Your rooms will appear here" + CTA.
    Empty search: icon + "Nothing found. Try something else."
    NEVER a blank white/black screen. That's a bug feeling.

  RULE AF-10 · TOUCH STATES ARE VISIBLE
    Every tappable item: has a pressed state (background flash, 100ms).
    --bg-surface-2 flash on press is enough. No scale needed on list items.
    (Scale is for buttons. Flash is for list items. Distinguish them.)

  RULE AF-11 · SYSTEM FONT FALLBACK READY
    If Inter doesn't load (slow network), system font (-apple-system,
    BlinkMacSystemFont) takes over with ZERO layout shift.
    Sizes are defined in px, not rem, to match system font metrics.

  RULE AF-12 · NO HORIZONTAL SCROLL ON MAIN LISTS
    Horizontal scrolling carousels = websites.
    EXCEPTION: Only Mood Rooms (horizontal scroll is intentional, compact).
    Everything else: vertical, full-width.

  ── 6.3 · THE ORBIT ROOM LIST ITEM (Specific Design) ─────────────────────────

  OLD (Telegram clone):
  ┌─────────────────────────────────────────────────────────────────────┐
  │ [icon 40px] │ Room Name (bold)          │ 2m        │
  │             │ last_user: message text   │ 🔵12       │
  │             │ 312 online                │           │
  └─────────────────────────────────────────────────────────────────────┘

  NEW (ORBIT native):
  ┌─────────────────────────────────────────────────────────────────────┐
  │ [icon 40px]  Room Name                               2m            │
  │              last message preview — truncated...      ●12          │
  │              ○ 312 live                                             │
  └─────────────────────────────────────────────────────────────────────┘

  Difference seems small — but these are the critical changes:
    • Icon: 40px, 12px radius (NOT circle). Rooms are not people. People =
      circle. Rooms = rounded square (like apps on your home screen).
    • Room name: 16px, weight 600, --text-primary
    • Message preview: 13px (caption size), --text-secondary, max 1 line
    • "312 live": caption, --text-tertiary. The ○ is a tiny 6px circle in
      --success color. Not "312 online" in green. Just tiny green dot + number.
    • Timestamp: 11px, --text-tertiary (same as always — this is fine)
    • Unread badge: --danger filled circle. NOT blue. Blue = read receipt.
      Red = unread. Don't copy Telegram's blue.
    • Left padding: 20px. Right padding: 20px.
    • Item height: 68px max.
    • Divider: 1px --border-subtle, starts at icon's right edge (not full width)
      — like iOS Messages app. This is a subtle app-native touch.

  ── 6.4 · SPOTLIGHT LIVE (Inline Treatment) ──────────────────────────────────

  Instead of a separate banner card (website widget), Spotlight Live becomes
  the FIRST item in the room list, but with a special visual treatment:

  ┌─────────────────────────────────────────────────────────────────────┐
  │ [●LIVE]  Gaming Lounge                         38 min left          │
  │          Spotlight · Bidding now               Bid →               │
  └─────────────────────────────────────────────────────────────────────┘

  - Background: --bg-surface-1 (NOT a separate card)
  - Left edge: 3px --warning (amber) vertical stripe
  - "●LIVE" pill: --warning color, caption size, no border
  - "Bid →" right side: --accent text, not a button

================================================================================
  PART 12 — 10 NON-NEGOTIABLES (This Navigation Layer)
================================================================================

  ── 1. FOUR TABS. EXACTLY FOUR. ───────────────────────────────────────────────
  No one adds a 5th tab and calls it an "improvement."
  Bazaar goes inside Discover. Full stop.

  ── 2. ACTIVE TAB = TOP LINE INDICATOR ONLY ──────────────────────────────────
  No pill background, no color fill, no box around active tab.
  A 2px line at the top of the tab item. That's it.
  Telegram does it. ORBIT does it better (2px vs Telegram's color fill).

  ── 3. ROOM LIST ITEMS = 68PX MAX HEIGHT ─────────────────────────────────────
  No website-card-style padding. Dense. Scannable. Native.

  ── 4. NO NEON CARD BACKGROUNDS IN MOOD ROOMS ────────────────────────────────
  All mood room cards: --bg-surface-1. Left stripe: max 3px.
  Violating this makes the entire Discover screen look like a children's app.

  ── 5. "YOU" ROW IN LEADERBOARD = ACCENT-SOFT ONLY ───────────────────────────
  Not solid blue. Not blue with border. Accent-soft (10% opacity) + 2px line.
  The blue whispers. It does not shout.

  ── 6. SECTION HEADERS = OVERLINE OR NOTHING ─────────────────────────────────
  "GROUP ROOMS" → gone.
  "MOOD ROOMS" → tiny 11px overline max, or removed.
  "WEEKLY CHALLENGES" → "This Week" as overline.
  All-caps bold section headers = web design. Not here.

  ── 7. BAZAAR FAB = ONLY ON BAZAAR VIEW ──────────────────────────────────────
  The "+ Post Gig" FAB appears ONLY when user is on Bazaar filter in Discover.
  It disappears on all other views.
  Persistent FABs that aren't contextual = clutter.

  ── 8. TIER LABELS = SUBTLE PILLS ONLY ───────────────────────────────────────
  "LEGEND" is NOT an orange bordered badge.
  "CHAMPION" is NOT a blue gradient pill.
  They are quiet --bg-surface-2 pills with --text-secondary text.
  The USER knows they're Legend. The app doesn't need to scream it.

  ── 9. ROOM ICONS = ROUNDED SQUARES (NOT CIRCLES) ────────────────────────────
  Circles = people (avatars). Rounded squares = rooms/groups.
  This single distinction makes it immediately clear: room ≠ person.

  ── 10. BUILD DISCOVER FIRST, ROOMS SECOND ───────────────────────────────────
  Discover tab is the most complex (4 filter states, mood rooms,
  challenges, bazaar). Build it first, get it right.
  Then port the list item design system to Rooms.
  The list item design is the same pattern — build once, use everywhere.
